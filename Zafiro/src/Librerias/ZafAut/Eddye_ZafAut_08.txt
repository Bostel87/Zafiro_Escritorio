v0.5: (Eddye Lino - 22/Oct/2010)
----
Se corrigieron los métodos:

Además se agregaron 2 columnas en el JTable para mostrar: "Fecha de vencimiento del cheque" y "Valor del cheque".


v0.6: (Eddye Lino - 27/Jul/2011)
----
Se modificaron los métodos: 
    1) cargarVenConCli()
    2) cargarVenConLoc()
En ambos métodos se modificó la sentencia SQL para que en lugar de validar con la fecha de vencimiento se valide
con la fecha actual.Se reemplazó a2.fe_ven por " + objParSis.getFuncionFechaHoraBaseDatos() + "
Antes:
    strSQL+=", -(a2.mo_pag+a2.nd_abo+(CASE WHEN a2.nd_monChq IS NULL THEN 0 ELSE (CASE WHEN a2.fe_ven<a2.fe_venChq THEN 0 ELSE a2.nd_monChq END) END)) AS nd_pen, a2.fe_venChq, a2.nd_monChq";
Después:
    strSQL+=", -(a2.mo_pag+a2.nd_abo+(CASE WHEN a2.nd_monChq IS NULL THEN 0 ELSE (CASE WHEN " + objParSis.getFuncionFechaHoraBaseDatos() + "<a2.fe_venChq THEN 0 ELSE a2.nd_monChq END) END)) AS nd_pen, a2.fe_venChq, a2.nd_monChq";
Con este cambio se logra que si la factura es del 15/Abr/2011, el cheque tiene fecha de 21/Abr/2011 y hoy es 18/Abr/2011
sólo se considere dicha factura en el control el 15, 16, 17, 18, 19 y 20 de abril porque el cheque es del 21. Es decir,
si hoy fuera 21 ya no se consideraría dicha factura en el control de ventas de contado.
